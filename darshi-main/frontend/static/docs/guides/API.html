Reports API

POST /api/v1/report

Create a new report with images and location data

  • Authentication: Optional (JWT Bearer token)
  • Rate Limit: 10 reports/hour (anonymous) | 50 reports/hour (authenticated)
  • Response Code: 201 Created

Request (multipart/form-data)

FieldTypeRequiredDescription
filesList[UploadFile]Yes1-5 image files (max 10MB each) - JPEG, PNG, WebP
locationstringYesHuman-readable location or coordinates (e.g., ‘28.7041,77.1025’ or ‘Delhi’)
titlestringYesReport title (max 200 characters)
descriptionstringNoDetailed description (max 1000 characters)
usernamestringNoAnonymous username (auto-generated if not provided)

Response

Notes

  • Background task processes AI verification asynchronously
  • Duplicate detection using dHash (perceptual hashing)
  • Images optimized to WebP and JPEG formats
  • Geohashing for proximity queries (~4.9km grid)
  • Reverse geocoding for human-readable addresses
  • Dynamic rate limiting: 5x more for authenticated users

Authentication API

POST /api/v1/auth/register

  • Authentication: None
  • Rate Limit: 5/hour (prevents mass account creation)
  • Response Code: 201 Created

Request (application/json)

Response

Notes

  • Username is required and must be unique
  • At least one of email or phone required

Response

Notes

  • Username field accepts username, email, or phone number
  • JWT token includes user_type='citizen'
  • Token expiration: 30 days
  • Brute-force protection via rate limiting

Authentication Schemes

JWT Bearer Token

  • Type: HTTP Bearer
  • Scheme: Bearer
  • Location: Authorization header
  • Format: Authorization: Bearer {token}
  • Token Type: JWT
  • Expiration: 30 days
  • Token Types:
    • user_type='citizen' for user tokens
    • user_type='admin' for admin tokens
  • Security: Admin tokens cannot be used for citizen endpoints and vice versa

Rate Limiting

Dynamic Limits

  • Report Creation:
    • Authenticated: 50 reports/hour
    • Anonymous: 10 reports/hour
    • Authenticated users get 5x more quota

Important Security Notes

  1. CSRF Protection: All OAuth endpoints use state parameter validation
  2. IDOR Prevention: Username/email extracted from JWT token, not user input
  3. User Enumeration: Generic error messages on login/registration failures
  4. Brute Force: Rate limiting on auth endpoints (3-10/hour)
  5. Admin Access: Separate user_type claim distinguishes citizen vs admin tokens
  6. Input Validation: All inputs sanitized via sanitize_form_data()
  7. PII: Passwords and tokens never logged
  8. Phone Verification: Phone changes require re-verification
  9. Password Security: bcrypt hashing with salt
  10. Email Verification: Verification tokens are time-limited
