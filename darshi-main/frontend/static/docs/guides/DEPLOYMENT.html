Prerequisites

Required Accounts & Tools

  1. Azure Account

    • Active Azure subscription
    • Access to Azure Portal
    • Budget: ~$50-70/month for 16GB VPS

  2. Cloudflare Account

    • Domain registered (or transferred to Cloudflare)
    • R2 Storage enabled
    • Pages access

  3. GitHub Account

    • Repository for code
    • GitHub Actions enabled

  4. Local Development Tools

    • SSH client: For VPS access
    • Docker: Install guide
    • Python 3.11+: For backend development
    • Node.js 20+: For frontend development
    • Git: For version control

  5. API Keys & Credentials

    • Gemini API Key: Get from Google AI Studio
    • OAuth Credentials: Google, GitHub, Facebook (optional)

Cost Estimates (Monthly)

Based on moderate usage (1000 reports/month):

  • Azure VPS (16GB RAM, 4 vCPU): ~$50-70
  • Cloudflare R2 Storage: ~$0-5 (first 10GB free)
  • Cloudflare Pages: Free (unlimited requests)
  • Domain (if purchasing): ~$10-15/year
  • Gemini API: Free tier (15 requests/min)

Total: ~$50-75/month for production workload

Components

  1. Frontend (Cloudflare Pages)

    • Static site generated with SvelteKit
    • Global CDN distribution
    • Automatic HTTPS
    • Zero config deployment

  2. Backend (Azure VPS + Docker)

    • FastAPI application
    • PostgreSQL 15 with PostGIS
    • Redis 7 with AOF persistence
    • Prometheus for metrics
    • Nginx reverse proxy

  3. Storage (Cloudflare R2)

    • S3-compatible object storage
    • Image hosting with CDN
    • No egress fees

  4. CI/CD (GitHub Actions)

    • Automated testing
    • Zero-downtime deployments
    • Container health checks

Azure VPS Setup

Step 1: Create Azure VM

  1. Go to Azure Portal

  2. Click Create a resource → Virtual Machine

  3. Configure:

    • Resource Group: Create new darshi-prod
    • VM Name: darshi-vps
    • Region: Choose closest to target users (e.g., Southeast Asia)
    • Image: Ubuntu Server 22.04 LTS
    • Size: Standard_D4s_v3 (4 vCPU, 16GB RAM)
    • Authentication: SSH public key (generate if needed)
    • Inbound Ports: SSH (22), HTTP (80), HTTPS (443)

  4. Click Review + Create → Create

  5. Wait for deployment (2-3 minutes)

  6. Note down the Public IP address (e.g., 20.193.150.79)

Storage Setup (Cloudflare R2)

Step 1: Create R2 Bucket

  1. Go to Cloudflare Dashboard
  2. Navigate to R2 Object Storage
  3. Click Create bucket
  4. Bucket name: darshi-reports
  5. Location: Automatic (global CDN)
  6. Click Create bucket

Step 2: Generate R2 API Credentials

  1. Go to R2 → Manage R2 API Tokens
  2. Click Create API Token
  3. Token name: darshi-backend
  4. Permissions: Object Read & Write
  5. Click Create API Token
  6. Copy the following:
    • Access Key ID
    • Secret Access Key
    • Endpoint URL (e.g., https://<account-id>.r2.cloudflarestorage.com)

Option A: Automatic (GitHub Integration)

  1. Go to Cloudflare Pages
  2. Click Create a project → Connect to Git
  3. Select repository: Sarthak-Sidhant/darshi
  4. Configure build:
    • Build command: cd frontend && npm run build
    • Build output directory: frontend/build
    • Root directory: /
  5. Environment variables:
    • VITE_API_URL: https://api.darshi.app
  6. Click Save and Deploy

Option B: Manual

Step 3: Configure Custom Domain

  1. Go to Pages project → Custom domains
  2. Click Set up a custom domain
  3. Enter: darshi.app
  4. Cloudflare will auto-configure DNS
  5. Wait for SSL certificate (2-5 minutes)

Security Checklist

  • Firewall configured (UFW)
  • SSH key authentication only (no password)
  • Non-root user created (darshi)
  • Docker containers run as non-root
  • SSL certificates installed and auto-renewing
  • Strong PostgreSQL password
  • Environment variables secured in .env
  • Rate limiting enabled in Nginx
  • Security headers configured
  • Regular backups scheduled
  • Monitoring set up (Prometheus)
  • Cloudflare proxy enabled (DDoS protection)
